# Project Map

## Repository overview

This repository is a Python CLI tool that transforms SVG artwork into printable A4 paint-by-numbers PDFs.
The current architecture is intentionally small but concentrated: most business logic lives in one script.

## Top-level map

- `svg_to_paint_by_numbers_pdf.py`
  - Main application entrypoint and main implementation file.
  - Owns CLI parsing, SVG ingestion, geometry processing, mystery fragmentation, label placement, rendering, progress reporting, profiling, and batch execution.

- `README.md`
  - Main user documentation.
  - Describes install steps, CLI usage, options, pipeline behavior, and limitations.

- `requirements.txt`
  - Runtime Python dependencies: `reportlab`, `shapely`, `svgpathtools`.

- `fonts/`
  - Stores font assets required for PDF label rendering.
  - Current critical asset: `fonts/Montserrat-Regular.ttf`.

- `patterns/`
  - Stores SVG patterns used by mystery/obfuscated output mode.
  - Current reference asset: `patterns/pattern.svg`.

- `inputs/`
  - Recommended folder for batch processing input SVG files.

- `output/`
  - Default destination for single-file PDF runs.
  - Generated content only.

- `pdf-output-*`
  - Timestamped batch output folders created under the input directory.
  - Generated content only.

- `.resources/`
  - Local resources and historical/generated data.
  - Not core source code.

- `.opencode/`
  - Agent memory, architectural maps, runbooks, and maintenance rules for future prompts.

- `scripts/`
  - Small maintenance utilities for repo-aware workflows.
  - Current utility: `scripts/generate_python_map.py` refreshes the autogenerated AST inventory inside `.opencode/python-map.md`.

## Runtime relationships

### Inputs

- Source SVG comes from a single file path or a directory of `.svg` files.
- Font file is loaded from `fonts/Montserrat-Regular.ttf` unless overridden with `--font-path`.
- Optional mystery fragmentation pattern comes from `patterns/pattern.svg` or another SVG passed with `--mystery-pattern`.

### Outputs

- Single-file mode writes `output/<name>_paint_by_numbers.pdf` unless `--output` is set.
- Batch mode creates `pdf-output-<timestamp>/` inside the input directory and writes one PDF per SVG.
- `--test` also writes profiling logs next to each produced PDF.

### External libraries and why they matter

- `svgpathtools`
  - Parses SVG path data and curve segments.

- `shapely`
  - Converts sampled SVG geometry into polygons and lines.
  - Powers intersections, buffering, validation, polylabel placement, and spatial indexing.

- `reportlab`
  - Generates final A4 PDF output, labels, legend, and line work.

## Stage-by-stage architecture

1. CLI stage
   - Parse user arguments and choose single-file or batch mode.

2. SVG ingestion stage
   - Parse XML, collect supported drawable elements, resolve style inheritance, and compute a usable view box.

3. Zone construction stage
   - Convert fills and optional strokes into polygons that can receive labels.

4. Palette normalization stage
   - Normalize the nearest dark color to pure black and derive sorted color labels.

5. Mystery stage
   - Optionally load a second SVG pattern, fit it to the drawing, and split zones by pattern cells.

6. Layout and rendering stage
   - Fit the drawing to A4, render grayscale outlines, render optional mystery boundaries, place labels, and draw the legend.

7. Diagnostics stage
   - Track per-stage timings, live progress, and optional profiling logs.

## Cross-file dependencies

- `README.md` depends on the CLI behavior implemented in `svg_to_paint_by_numbers_pdf.py`.
- `fonts/Montserrat-Regular.ttf` is required by the renderer and startup font registration.
- `patterns/pattern.svg` is consumed by the mystery pattern loader when requested.
- `.gitignore` and `.dockerignore` intentionally ignore generated outputs and `.env`.
- `.opencode/*.md` should mirror the current implementation and be refreshed whenever behavior changes.
- `scripts/generate_python_map.py` depends on `svg_to_paint_by_numbers_pdf.py` and `.opencode/python-map.md` and only rewrites the marker-delimited autogenerated block.

## Non-source artifacts to ignore during analysis

- `output/`
- `pdf-output-*`
- `__pycache__/`
- `.resources/` generated PDFs and logs
- local environment files like `.env`

## Current architecture note

The repository is still effectively a monolith around `svg_to_paint_by_numbers_pdf.py`.
If the script is ever split into modules, this map should be reorganized around packages such as `cli`, `svg`, `geometry`, `labels`, `render`, and `diagnostics`.
